Idea 5

AI Systems Documentation & Playbook Generator

An AI assistant that turns messy reality—code, configs, tickets, Slack threads—into clean documentation, runbooks, and onboarding guides for technical teams.

Instead of “tribal knowledge” and outdated wiki pages, teams get living docs that stay in sync with how the system actually works today.

Problem

Every engineering / data team knows the pain:

  • Critical systems are understood by 1–2 “heroes” who get pinged for everything.
  • Docs are scattered across README files, Confluence, Notion, and old Google Docs.
  • Onboarding new engineers is slow and fragile; outages are harder to debug than they need to be.

Everyone agrees documentation matters, but it’s always postponed “until later”.

Solution

An AI system that continuously ingests signals from your stack and generates the documentation and playbooks you wish you had:

  • Connects to Git repositories, issue trackers, CI logs, and chat (Slack / Teams).
  • Builds a knowledge graph of services, dependencies, and key workflows.
  • Generates human-readable:
    • System overviews and architecture descriptions.
    • Runbooks for common incidents (“DB latency”, “queue backlog”).
    • Onboarding guides tailored to specific roles (backend, data, ops).
  • Keeps docs “fresh” by proposing updates when code or infra changes.

Engineers review and accept changes, keeping humans in control of the source of truth.

Example Workflow

  • Connect GitHub / GitLab, Jira / Linear, Slack, and your infra provider.
  • AI maps services, main flows, and recurring incidents.
  • It generates a “System Overview” and 3–5 initial runbooks.
  • Team reviews, edits, and approves with change tracking.
  • Whenever significant changes land (new service, big refactor), the system suggests doc updates or new playbooks.

Why Us

With backgrounds in materials, mechanics, physics, and full-stack SaaS, we’re used to documenting complex systems for students, reviewers, and collaborators. This idea turns that documentation habit into a product for engineering teams.

Snapshot

Status: Idea → MVP concept.

Target users:

  • Early-stage SaaS startups with growing engineering teams.
  • Scale-ups with microservices and on-call rotations.
  • Data / ML teams with complex pipelines and handover issues.

Business Model

  • SaaS pricing per engineering seat or per project.
  • Higher tiers for SSO, compliance exports (SOC2 evidence), and audit logs.
  • Professional services: “doc sprints” where you co-create with larger teams.

Tech Stack (Draft)

  • Connectors for GitHub / GitLab, Jira / Linear, Slack, cloud providers.
  • Backend: Python / Node.js with a Postgres DB and a vector store.
  • LLMs for summarisation, code understanding, and doc generation.
  • Frontend: React / Next.js app or embedded widgets in existing docs tools.
